# 과제 1

## 난이도 선택 가이드

### Easy

- 테스트를 처음 작성해보시는 분들을 위한 과제입니다. 가벼운 유틸함수, 훅 기반 단위 테스트를 작성해보면서 컴포넌트를 개선해보세요.
- 모든 테스트케이스, 설정이 제공됩니다.
- 작성을 해보고 피어 리뷰와 멘토들의 피드백을 받아보세요!
- **하지만, 모두 통과를 하더라도 합격을 받을 수는 없습니다. 가능하면 이 난이도를 선택하지 마세요**

### Medium

- Easy에 비해 추가로 작성할 여러 통합 테스트가 존재합니다.
- PR template에 있는 여러 질문에 답을 해보면서 문제를 풀어보세요
- **Best가 불가능합니다**

### Hard

- Medium과 동일한 TC를 작성하지만, 과제를 작성하기 위한 여러가지 설정을 제공드리지 않습니다.
- PR template에 있는 여러 질문에 답을 해보면서 문제를 풀어보세요!
- **적절하게 목적에 맞게 작성하셨을 경우 Best가 가능합니다.**

## 과제 셀프회고

딱 하루 일정이 있어서 못 했는데 과제 제출 기간이 생각보다 너무 촉박 했다.
내가 코드를 작성하고 공부하는 부분에 있어서 너무 느린게 아닌가 실감했다.
누구는 출퇴근 하며 해도 미리 끝내는 모습을 보면 스스로와 수준 차이가 느껴졌다.
=> 그렇다면 두배로 더 공부하는게 맞다.

내가 고민하고 거슬렸던 부분은 남들의 설명과 코드를 보니 삽질 하고 있던 것들이 많았다.
고물상 처럼 잡다한 지식으로 풀려고 하니 이상한 계열로 가버려서 다시 돌아오는 시간이 걸리는거 같다.
=> 좀더 확실하게 정보를 알고 진행 하는 버릇을 들이자.

그나마 사전 스터디로 어느정도 테스트 코드의 개념과 실습이 있었기에 망정이지 없었으면 하루종일 질문만 하며 다니고
과제도 끝까지 못할뻔 했다.
=> 준비의 중요함을 깨닫고 토요일 되기 전이라도 강의와 리팩토링 책을 달고 살아야겠다.

그래도 과제 끝까지 해서 다행이다.
=> 다음에는 과제 Hard 잡고 첫날부터 밤새 하고싶다.

### 코드 품질

--- 결론 적으론 많이 아쉬운 품질
- 시간에 쫒기다 보니 리팩토링 안한 부분이 눈에 밟힌다.
- 리팩토링 하려고 한 부분만 5곳이 넘는데 해보지 못해 아쉽다.
- 그나마 초반인 unit 폴더 부분은 다른 부분보다 가독성 있게 작성되어 있다.

--- 리팩토링이 필요한 부분
- events 객체 생성 부분
- 날짜 계산 helper 함수
- 테스트용 네트워크 에러 부분
- 테스트 코드 반복되는 훅 선언 부분
- App.tsx 컴포넌트 화

### 학습 효과 분석

--- 가장 큰 배움이 있었던 부분
- 테스트 코드에서 HTML 요소 찾아가는 부분
- 테스트 코드는 최적화가 아니라 가시적이게 잘 정리하는 것도 중요하다는 것

--- 추가 학습이 필요한 영역
- 타이머 등 JSDOM 내부에서 이뤄질 현상을 머리 속으로 그리는 부분
- 전체적인 페이지 이해도

--- 실무 적용 가능성
- 매우 높음


### 과제 피드백

--- 과제에서 모호하거나 애매했던 부분
- 테스트 코드 렌더링을 위한 MUI 태그 찾아가기 어려운...
- 날짜 함수들이 어디는 JS 미국식, ISO 식으로 되어 있어서 혼동 되었다.
- 객체 이름이 event 라 DOM 이벤트랑 햇갈렸다.

--- 과제에서 좋았던 부분
- 테스트 코드 작성해 보라고 하면 해볼 수 있을 만큼의 코드 작성 양

## 리뷰 받고 싶은 내용

- unit 부분에서 테스트 코드가 코치님께서 보시기에 가시적인지 확인 해주시면 좋겠습니다.
- 최대한 가시적이게 변수 명을 지어봤지만 함수나 비교부분(expect) 에 직접 값을 써 넣는게 더 좋은지, 변수로 지금처럼 빼서 변수명 잘 지어주는게 좋은지 조언 부탁드립니다.
- 테스트 helper 함수 만들때 함수니까 효율적이지만 복잡하게 만들어도 되는지, 아니면 그것도 확인 해야 하기 때문에 단순화 해야 하는지 알고 싶습니다.
